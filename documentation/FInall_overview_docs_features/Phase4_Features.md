## 1Ô∏è‚É£ ROLE DEFINITION ‚Äî FACULTY PORTAL

### Role: **FACULTY**

#### Purpose

The Faculty role is responsible for **academic delivery** using the content and packages provided to the college.

Faculty members **teach**, **structure courses**, **assess students**, and **track learning progress**, but **do not manage institutions or content ownership**.

---

### Responsibilities (High-Level)

* Create and manage courses
* Design learning flows
* Assign courses to students
* Conduct assessments
* Track and analyze student performance

---

### Explicit Boundaries

Faculty **CANNOT**:

* Create or modify publisher content
* Modify content packages
* Access other colleges‚Äô data
* Manage students at institution level
* Change platform or college settings

## 2Ô∏è‚É£ COURSE CREATION & MANAGEMENT

### 2.1 Course Creation

Faculty can:

* Create courses using assigned content packages
* Define course metadata:

  * Course title
  * Description
  * Academic year
  * Department
* Save courses as Draft
* Publish courses for student access

Rules:

* Courses can only use content from assigned packages
* A course must belong to the faculty‚Äôs college
* Draft courses are invisible to students

---

### 2.2 Learning Flow Design

Faculty can:

* Create learning flow steps
* Define step sequence
* Assign learning units to steps
* Mark steps as:

  * Mandatory
  * Optional
* Define prerequisite chains between steps

Rules:

* Prerequisites must form a valid sequence
* Circular dependencies are not allowed
* Locked steps remain inaccessible until prerequisites are met

---

## 3Ô∏è‚É£ CONTENT USAGE (FACULTY SCOPE)

Restrictions:

* Cannot upload new learning units
* Cannot edit learning unit content
* Cannot access content outside package scope
